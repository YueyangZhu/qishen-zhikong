# 审核流程统一 + 测试合同生成 + 样例风险数量错开

## 一、问题总结

1. **审核流程不一致**：上传文件走弹窗（DeepSeek识别）→ 直接进详情页；样例合同走进度页面（7阶段）→ 进详情页。用户希望都走进度页面。
2. **表格/图片未展示**：上传带表格的PDF合同，详情页未展示表格。用户希望生成几份带表格/图片的本地合同文件供测试。
3. **样例风险数量都一样**：6份样例合同每份都是10个风险，看起来不真实，需要错开。

## 二、方案

### 改动1：审核流程统一到进度页面

**核心思路**：上传文件也走"创建草稿任务 → startReview → 跳进度页"流程。进度页检测任务是否为真实AI模式，真实AI模式时调用 `runFullAIReview`，onProgress 映射到7阶段更新任务，完成后调 `createTaskWithAIResult`。

**文件对象传递**：File 对象无法存 localStorage（太大），用 zustand 内存 store 暂存（不持久化），跳转后进度页从 store 取。

**具体改动**：

#### 1.1 新建 `src/store/useRealAIStore.ts`（zustand 内存 store）
```typescript
interface RealAIState {
  file: File | null;
  options: { contractType?: string; myRole?: 'buyer' | 'seller'; reviewFocus?: string[]; reviewNote?: string } | null;
  taskId: string | null;
  running: boolean;  // 防止重复执行
  set: (file: File, options, taskId) => void;
  clear: () => void;
}
```

#### 1.2 修改 `src/services/reviewService.ts`
- `ReviewTask` 类型新增字段 `realAI: boolean`（标记是否真实AI审核，默认 false）
- 新增方法 `startRealAIReview(id, user, file, options)`：
  - 调用 `startReview` 设置任务为 parsing 状态
  - 设置 `task.realAI = true`
  - 把 file 和 options 存入 `useRealAIStore`
- 新增方法 `updateRealAIStage(id, stage, progress)`：更新任务的 currentStage 和 progress（供进度页 onProgress 回调用）
- 新增方法 `completeRealAIReview(id, aiResult, user)`：封装 `createTaskWithAIResult` 逻辑 + 清理 store

#### 1.3 修改 `src/pages/ReviewNewPage.tsx`
- `handleStartReview` 中 `useRealAI` 分支改为：
  ```typescript
  // 创建草稿任务（复用 draftId）
  let taskId;
  if (draftId) { await reviewService.updateTask(draftId, buildInput(), currentUser); taskId = draftId; }
  else { const task = await reviewService.createTask(buildInput(), currentUser); taskId = task.id; }
  // 启动真实AI审核（存file到store + startReview）
  await reviewService.startRealAIReview(taskId, currentUser, file!.rawFile!, { contractType, myRole, reviewFocus, reviewNote });
  message.success('已发起 AI 审核，正在解析合同...');
  setTimeout(() => navigate(`/reviews/${taskId}/progress`), 600);
  ```
- 删除原 `runFullAIReview` 弹窗逻辑和 `setAiProgress` 状态

#### 1.4 修改 `src/pages/ReviewProgressPage.tsx`
- 新增真实AI检测：`useEffect` 中检测 `result.task.realAI`
- 真实AI分支（用 `useRef` 防止重复执行）：
  ```typescript
  const realAIRun = useRef(false);
  useEffect(() => {
    if (!result?.task.realAI || realAIRun.current) return;
    realAIRun.current = true;
    const { file, options } = useRealAIStore.getState();
    runFullAIReview(file, options, async (p) => {
      // 映射3阶段到7阶段
      const stageMap = { parse: 'parse', extract: 'extract', review: 'ai', done: 'result', error: 'parse' };
      await reviewService.updateRealAIStage(taskId, stageMap[p.stage], p.progress);
      fetchProgress(); // 刷新UI
    }).then(async (aiResult) => {
      await reviewService.completeRealAIReview(taskId, aiResult, currentUser);
      fetchProgress(); // done=true 触发跳转
    }).catch((e) => {
      // 标记任务失败
    });
  }, [result?.task.realAI]);
  ```
- 时间模拟分支（样例）：保持现有逻辑不变

#### 1.5 修改 `src/types/index.ts`
- `ReviewTask` 接口新增 `realAI?: boolean` 字段

### 改动2：生成4份带表格/图片的测试合同文件

**文件位置**：`backend/test_contracts/` 目录（新建）

**生成脚本**：`backend/scripts/gen_test_contracts.py`（新建，运行后生成4份文件）

**4份文件**：
1. `测试合同_含表格.pdf` - 含付款计划表、交付清单表（2个表格，无图片）
2. `测试合同_含图片.pdf` - 含公司logo图片、签署盖章图（2个图片，无表格）
3. `测试合同_表格图片.docx` - 含1个表格+1个图片
4. `测试合同_综合版.docx` - 含2个表格+2个图片（综合测试）

**生成依赖**：reportlab（PDF）、python-docx（DOCX）、Pillow（图片生成）

**脚本逻辑**：
- 用 Pillow 生成简单的占位图片（公司logo、印章图，纯色块+文字）
- 用 reportlab 生成 PDF：插入段落文本 + Table 对象 + Image 对象
- 用 python-docx 生成 DOCX：插入段落 + add_table + add_picture

### 改动3：6份样例风险数量错开

**目标分布**（覆盖各种风险类型，等级不全要求一致）：
| 样例 | 风险数 | high | medium | low | notice |
|------|--------|------|--------|-----|--------|
| 1 办公设备 | 8 | 2 | 3 | 2 | 1 |
| 2 IT运维 | 9 | 2 | 4 | 2 | 1 |
| 3 云服务器 | 10 | 3 | 4 | 2 | 1 |
| 4 人力外包 | 11 | 3 | 4 | 3 | 1 |
| 5 装修工程 | 12 | 3 | 5 | 3 | 1 |
| 6 设备融资租赁 | 13 | 4 | 5 | 3 | 1 |

**改动文件**：`src/mock/sampleContracts.ts`
- 每份合同的 risks 数组按上表调整数量
- 风险类型尽量多样：subject/amount/payment/delivery/acceptance/warranty/breach/termination/ip/confidentiality/data_security/dispute/term
- 新增/删除风险时，必须确保 originalText 是段落 text 的连续子串，startPosition/endPosition 精确

## 三、不改动

- `runFullAIReview` 函数本身（apiClient.ts）保持不变，进度页调用它
- 样例合同的 Mock 进度逻辑（基于时间戳）保持不变
- PDF/DOCX 解析逻辑（pdf_service.py）保持不变（表格图片提取已实现）
- ContractTextView 表格图片渲染保持不变（已实现）

## 四、验证步骤

1. `npx tsc --noEmit` TypeScript 编译通过
2. `python -m py_compile` Python 语法验证
3. 运行 `python backend/scripts/gen_test_contracts.py` 生成4份测试文件
4. 本地验证：
   - 上传测试合同 → 应跳进度页（7阶段）→ 完成跳详情页
   - 详情页应展示表格和图片
   - 样例合同 → 走进度页（时间模拟）→ 完成跳详情页
   - 6份样例风险数量分别为 8/9/10/11/12/13

## 五、风险与兼容

- `realAI` 字段可选，旧数据无此字段时默认 false，走时间模拟逻辑
- File 对象在 store 中不持久化，刷新页面会丢失（真实AI任务刷新后无法恢复，标记为失败并提示重新上传）
- 4份测试合同文件放 backend/ 目录，不影响前端代码
