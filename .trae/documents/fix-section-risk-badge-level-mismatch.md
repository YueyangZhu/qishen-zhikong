# 修复章节标题风险标识与正文风险等级不一致

## Summary

当前在结构化视图中，body 段落的章节标题（`# 第三条 合同金额与付款方式`）旁边会显示一个风险等级 badge（如“中风险”），但正文里同一段落的具体风险高亮可能是其他等级（如高风险）。这种标题标识与正文标识不一致，容易让用户误以为该段落的风险等级就是 badge 所示等级。

用户期望标题的风险标识采用和正文风险高亮一致的视觉语言：字体颜色 + 背景色 + 下划线，而不是一个独立的等级 badge。同时，标题必须只按“自己身上实际存在的风险”来标识，不能因为正文里有更高等级的风险就把整个标题标红。

## Current State Analysis

### 关键文件

- `src/features/review/ContractTextView.tsx`
  - `ParagraphItem` 组件负责渲染 body 段落。
  - 当前在 `para.clauseNo && (...)` 的标题行末尾，会为该段落的每个风险渲染一个等级 badge（`中风险` / `高风险` 等）。
  - badge 颜色和正文里 `<mark>` 高亮颜色来自同一份 `RISK_LEVEL_MAP`，但当段落存在多个风险、或 badge 只展示了其中一个风险时，标题 badge 与正文具体高亮的等级可能不一致。
  - 正文风险高亮使用 `background` + `color` + `borderBottom` 的 inline `mark` 样式。

- `src/constants/index.ts`
  - `RISK_LEVEL_MAP` 定义了 `high / medium / low / notice` 的颜色、背景、边框等样式。

### 问题根因

1. 标题行使用 badge 形式展示风险等级，与正文使用的高亮形式不同。
2. 当段落存在多个不同等级风险时，标题 badge 可能只展示其中一个，导致与正文高亮等级不一致。
3. 用户视觉上会把标题旁的 badge 当成“该段落的风险等级”，而正文里实际高亮的可能是更高或更低的等级。

## Proposed Changes

### 修改文件：`src/features/review/ContractTextView.tsx`

#### 1. 去掉章节标题旁的等级 badge

- 删除 `para.clauseNo && (...)` 内部 `{highlights.length > 0 && (...)}` 的 badge 渲染逻辑。

#### 2. 用与正文 mark 一致的样式标识标题上实际存在的风险

- 在 `ParagraphItem` 中，沿用去重逻辑计算出的 `prefixLen`（即被去除的标题文本长度）。
- 将 `highlights` 分成两类：
  - **标题内风险**：`highlight.start < prefixLen`（风险原文落在标题区域内，即缺失/遗漏类规则定位到标题的情况）。
  - **正文风险**：`highlight.start >= prefixLen`（风险原文在正文区域）。
- 标题内风险使用自己的实际等级来渲染标题样式：
  - 如果标题内没有风险，标题行保持默认样式，**不标任何颜色**。
  - 如果标题内有一个或多个风险，取标题内风险中按 `RISK_LEVEL_MAP[level].rank` 最高的等级作为标题样式。
  - 标题样式与正文 `mark` 一致：`background` + `color` + `borderBottom` + `borderRadius` + `padding`。
- 点击带样式的标题行时，激活标题内最高等级风险的 `riskId`；如果标题内没有风险，点击标题行不触发激活。

#### 3. 保留正文风险高亮逻辑不变

- 正文里的 `<mark>` 风险高亮样式和交互保持原样。
- 去重章节标题前缀的逻辑保持原样，避免重复标题重新出现。

### 样式细节

- 标题行的风险样式应该和正文 `mark` 样式尽量一致：
  - 高风险：红字 + 浅红背景 + 红色下划线
  - 中风险：橙字 + 浅橙背景 + 橙色下划线
  - 低风险：绿字 + 浅绿背景 + 绿色下划线
  - 提示项：蓝灰字 + 浅蓝背景 + 蓝灰下划线
- 激活状态（activeRiskId 匹配该段落风险）时，可额外加 `boxShadow` 或 `fontWeight` 与正文 active 状态一致。

## Assumptions & Decisions

- **决策：标题只展示“标题区域内实际存在的风险”的最高等级，而不是整个段落的最高风险等级。**
  - 原因：用户明确反对“因为正文有高风险就把整个标题标红”。标题本身若有缺失/遗漏类风险（风险原文落在标题上），才按该风险等级标识；若标题上没有风险，即使正文有高风险，标题也不标颜色。
- **决策：标题风险标识复用正文 mark 的视觉语言（背景色 + 字体色 + 下划线）。**
  - 原因：用户明确要求“像正文内容的风险一样标注字体颜色和背景色划线”，统一视觉语言可消除误解。
- **决策：点击标题行激活代表风险。**
  - 原因：保持与正文风险高亮相同的交互习惯，点击后右侧风险明细滚动到对应风险卡片。

## Verification Steps

1. 启动本地前端开发服务器，打开任意有风险的审核任务详情页。
2. 检查结构化视图中各 body 段落标题：
   - 无风险的段落标题保持默认样式，无 badge、无高亮。
   - 有风险段落的标题使用对应最高风险等级的背景色 + 字体色 + 下划线。
3. 对比同一标题与正文里具体风险高亮的颜色：
   - 若风险原文就是标题本身，标题颜色应与该风险等级一致。
   - 若风险原文只在正文，标题不应标颜色，仅正文高亮。
4. 点击带颜色的标题，右侧风险明细应激活并滚动到对应风险卡片。
5. 运行 `npm run build` 确认 TypeScript 与构建无错误。
6. 运行 Playwright 自动化验证脚本（如 `scripts/verify_desktop.py`）确认三栏布局、滚动、风险顺序未受影响。
