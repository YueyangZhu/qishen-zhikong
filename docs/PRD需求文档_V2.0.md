## 一、产品概述

### 1.1 产品定位

**契审智控** 是一款面向企业采购合同审核场景的 AI 辅助审核平台。系统融合规则引擎与大模型语义分析，覆盖合同解析、字段抽取、风险识别、原文定位、修改建议、人工确认、法务复核、报告生成的完整闭环。

### 1.2 核心业务流程

```
上传合同 → 文档解析 → 字段抽取 → 规则引擎检查 → AI 语义审核
→ 风险原文定位 → 生成风险说明与修改建议 → 业务人员逐条处理
→ 提交法务复核 → 法务确认 → 生成审核报告 → 审核记录留档
```

### 1.3 当前阶段边界

| 已实现 | 暂未实现 |
|--------|----------|
| 本地可交互前端 Demo（12 个页面） | 真实大模型 API 接入 |
| 完整业务闭环演示 | 真实 Supabase 后端 |
| 高质量 Mock 数据层 + localStorage 持久化 | 真实鉴权（Mock 登录） |
| 端到端 23 步演示流程 | 扫描 PDF OCR |
| 浏览器打印导出 PDF | Word 红线修订 |
| 角色切换、数据重置 | 电子签章 / 复杂审批流 |

### 1.4 系统免责声明

> 本系统审核结果由 AI 辅助生成，仅供合同初审参考，不构成正式法律意见，最终结论应由专业人员确认。

免责声明在以下位置展示：
- 登录页底部黄色提示框
- 全局左侧导航底部
- 审核进度页底部
- 审核报告详情页底部

2. 填写审核信息
3. 确认并发起

- 文件类型限制：PDF / DOCX
- 文件大小限制：10MB

- 上传成功后显示文件卡片（名称、大小、删除按钮）

- 合同名称（必填）
- 合同类型（必选）
- 我方身份（甲方 / 乙方，必选）
- 相对方（必填）
- 所属部门（必选）
- 合同金额（数字输入，必填）
- 币种（CNY / USD / EUR）
- 审核重点（多选 Checkbox）：
  - 合同主体 / 付款条款 / 交付与验收 / 违约责任 / 知识产权 / 保密与数据安全 / 合同解除 / 争议解决

- 文件信息摘要

### 校验规则

- 文件格式非 PDF/DOCX：错误提示并阻止
- 文件大小超 10MB：错误提示并阻止
- 必填项未填：表单项红色提示
- 必须选择演示合同或上传文件后才能进入第二步

## P05 审核处理进度页

**路由**：`/reviews/:id/progress`  ｜  **截图**：![](screenshots/P05-审核进度.png)

### 页面元素

**顶部摘要卡**
- 当前阶段名称（如「正在抽取关键字段」）
- 阶段描述
- 整体进度百分比（28px 大字 + 蓝色渐变进度条）
- Sparkles AI 图标

**处理阶段卡**（Steps 垂直方向）

| 序号 | 阶段 | 说明 |
|------|------|------|
| 1 | 上传文件 | 文件已接收 |
| 2 | 解析文档 | OCR + 文本提取 |
| 3 | 识别合同结构 | 章节切分 |
| 4 | 抽取关键字段 | 金额、日期、主体等 |
| 5 | 执行风险规则 | 规则引擎匹配 |
| 6 | 执行 AI 语义审核 | 大模型分析 |
| 7 | 生成审核结果 | 整理输出 |

每阶段状态：`wait` / `processing` / `success` / `failed`

**底部操作区**
- 「模拟解析失败（演示用）」红色链接按钮
- 「预计处理时间约 10 秒」提示

**完成态**
- Result 成功组件
- 风险数量摘要
- 「进入审核详情」主按钮

### 交互逻辑

- 进度基于时间戳计算（`localStorage.data:reviewStarts` 记录启动时间）
- 刷新页面后基于 `Date.now() - start` 重新计算进度，不丢失状态
- 每阶段约 1.5 秒，全程约 10 秒
- 全部完成 1 秒后自动跳转 `/reviews/:id`
- 点击「模拟解析失败」将任务标记为 `failed` 状态
- 失败后显示错误 Result + 「重新审核」按钮

**字段表格**

| 列名 | 说明 |
|------|------|
| 字段名 | fieldLabel |
| 字段值 | fieldValue（可编辑） |
| 置信度 | confidence 百分比 + 颜色标签 |
| 来源原文 | sourceText（点击查看） |

- 字段未全部确认时，提交法务复核会有提示

## 五、核心业务状态机

### 5.1 审核任务状态机

```
                  ┌─────────────┐
                  │    draft    │ ← 保存草稿
                  └──────┬──────┘
                         │ 开始审核
                         ▼
                  ┌─────────────┐
                  │   parsing   │
                  └──────┬──────┘
                         │ 解析完成
                         ▼
                  ┌─────────────┐
                  │ ai_reviewing│
                  └──────┬──────┘
                         │ AI 审核完成
                         ▼
                  ┌──────────────┐
                  │pending_business│ ← 业务人员处理
                  └──────┬───────┘
                         │ 提交法务复核
                         ▼
                  ┌──────────────┐
                  │ pending_legal │ ← 法务复核
                  └──────┬───────┘
            ┌────────────┼────────────┐
            │退回         │完成        │失败
            ▼            ▼            ▼
   pending_business  completed     failed
```

### 5.2 风险项状态机

```
                  ┌──────────┐
                  │ pending  │
                  └────┬─────┘
            ┌──────────┼──────────┬──────────┐
            ▼          ▼          ▼          ▼
       accepted    edited     ignored   manual_review
            │          │          │          │
            └──────────┴──────────┴──────────┘
                       │ 恢复处理
                       ▼
                   pending

法务复核阶段：
   pending/edited → confirmed（法务确认）
```

### 5.3 提交法务复核校验规则

`checkCanSubmitForLegalReview(task, risks, force)` 校验：
1. 任务状态必须为 `pending_business`
2. 所有 `riskLevel === 'high'` 的风险必须 status ∈ `[accepted, edited, ignored, manual_review]`
3. 所有字段必须 `confirmed === true`
4. 校验失败返回 `{ canSubmit: false, reasons: [...] }`

## 六、数据一致性约束

### 6.1 统一数据源

所有页面数据来自 `src/services/db.ts` 的 localStorage 集合：

| 集合 | Key | 内容 |
|------|-----|------|
| users | `data:users` | 3 个演示账号 |
| tasks | `data:tasks` | 5 个预设审核任务 |
| risks | `data:risks` | 演示合同预埋的 18 个风险 |
| fields | `data:fields` | 每个任务 15 个抽取字段 |
| reports | `data:reports` | 已生成的报告 |
| rules | `data:rules` | 风险规则库 |
| auditLogs | `data:auditLogs` | 全部操作日志 |

### 6.2 数据联动规则

| 操作 | 联动效果 |
|------|----------|
| 风险处理（接受/编辑/忽略/转人工） | `db.recalcTaskStats` 重算任务 riskCount + riskLevelMax，列表与工作台同步 |
| 提交法务复核 | 任务状态 → pending_legal，列表状态变化 |
| 法务完成审核 | 任务状态 → completed，自动生成报告，报告列表出现新记录 |
| 法务退回 | 任务状态 → pending_business，业务人员可重新处理 |
| 字段编辑 | 字段 confirmed = true，写入审计日志 |
| 规则启停 | 规则状态变化，规则列表实时反映 |
| 所有操作 | 写入 `auditLogs`，审核记录页可查看 |

### 6.3 报告快照机制

- 报告生成时调用 `generateReportSnapshot(task, fields, risks, aiSummary, legalOpinion, conclusion)`
- 快照为不可变对象，存储在 `report.snapshot` 字段
- 即使后续风险变化，已生成报告的内容不会改变
- 重新生成报告会创建新版本（versionNo + 1）
