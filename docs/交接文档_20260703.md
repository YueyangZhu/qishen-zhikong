## 四、未解决的问题

### 高优先级

#### 1. 原文格式预览不完美

**问题描述**：
- **演示合同**（seed 初始化）：`_text_to_html` 将纯文本粗暴转为简单 HTML，无表格/样式/格式
- **用户上传的 DOCX**：`pdf_service.py` 的 `_generate_html` 用 mammoth 生成了 HTML，但用户反馈视觉效果与原文件不一致

**分析**：
- 演示合同没有真实 DOCX 源文件，所以 seed 只能用纯文本→HTML
- 上传的 DOCX 用 mammoth 转 HTML 后，样式（字体、行距、间距等）与 Word 打开不完全一致（mammoth 的特性限制）

**可能的修复方向**：
1. 为演示合同准备一份真实的 DOCX 样本文件放入 `backend/test_contracts/`，seed 时复制并 mammoth 解析
2. mammoth 支持自定义样式映射，可以调整 `convert_to_html` 的 `style_map` 参数优化视觉效果
3. 考虑使用 `docx-preview` 或类似前端库直接在浏览器渲染 DOCX（无需后端转换）

#### 2. DeepSeek API 余额不足/限流

**问题描述**：
- 当前 DeepSeek 账户余额已用完：`insufficient_quota`
- 经常触发 TPM 限流：`rate limit exceeded on dimension: tpm`

**现状**：已清空 `DEEPSEEK_API_KEY`，系统在 Mock 模式运行，预设数据可走通全流程
**修复**：充值后重新配置 API Key 即可，代码本身无 bug

#### 3. 本地后端启动后的验证

在本地 TRAE 中启动后需要：
1. 调用一次 `/api/data/seed` 初始化数据
2. 检查 `GET /api/data/documents/RVT-DEMO-003` 的返回中 `htmlContent` 非空
3. 上传一个测试 DOCX 文件测试原文预览

### 低优先级

- `pdf_service.py` 的 `_generate_html` 中 PDF 返回的是占位文字（`PDF 文件暂不支持原文格式预览`），如需 PDF 原文预览需额外实现
