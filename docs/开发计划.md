- 项目目录原为空，仅含 `reference/契审智控_AI采购合同审核平台_PRD_V1.0.docx`（PRD V1.0）。

## 二、产品范围

面向企业采购合同审核场景的 AI 初审与人工复核闭环工作台。本阶段为「本地可交互前端 Demo」，使用高质量 Mock 数据完成完整演示，刷新后数据保留，可稳定完成一次端到端采购合同审核演示。不接真实大模型/Supabase/鉴权，不部署/推送。

## 四、核心业务流程

```
上传合同 → 文档解析 → 合同字段抽取 → 规则引擎检查 → AI语义审核
→ 风险原文定位 → 生成风险说明和修改建议 → 业务人员逐条处理
→ 提交法务复核 → 法务确认 → 生成审核报告 → 审核记录留档
```

任务状态机：`draft → parsing → ai_reviewing → pending_business → pending_legal → completed`（任意阶段可 `failed`）。

## 五、核心数据对象

- Contract 合同 / ContractVersion 合同版本
- ReviewTask 审核任务 / ExtractedField 抽取字段
- RiskItem 风险项 / RiskAction 风险处理记录
- ReviewReport 审核报告 / RiskRule 风险规则 / AuditLog 审计日志 / User 用户
- 详见 `docs/数据模型.md`。

## 六、技术实施方案

- 路由：React Router 6 + 守卫（登录态校验、角色校验），刷新可访问。
- 状态：Zustand 管理登录态、当前任务缓存；Mock 数据层独立于 store，提供 `services/*` 业务服务。
- 持久化：`utils/storage.ts` 封装 localStorage，统一命名空间 `qszk:*`。
- 主题：`theme/` 集中 AntD ConfigProvider token + CSS 变量 Design Token。
- 图表：ECharts，数据从统一数据源计算，杜绝互相矛盾。
- Mock 审核：`services/reviewService.ts` 用 setTimeout 模拟七阶段进度，失败可重试。
- 预留接口：`services/*` 方法签名与 `docs/API设计.md` 一致，后续替换为真实后端不改调用方。

## 八、演示主流程（必须端到端联动）

1. 采购业务人员登录 → 工作台 → 新建审核
2. 上传演示合同 → 填写审核信息 → 发起 AI 审核
3. 查看进度 → 确认抽取字段 → 进入三栏详情
4. 点击风险卡定位原文 → 接受一条 / 编辑一条 / 忽略一条（填原因）/ 转人工一条
5. 提交法务复核 → 切法务 → 查看业务处理结果 → 填法务意见 → 选「建议修改后签署」→ 完成审核
6. 生成并查看报告 → 查看审核历史与时间轴
7. 返回工作台，统计数据同步更新
